# 4. Схема загрузки чатов

## Главное правило

Старые чаты целиком НЕ загружать на старте нового Project.

Почему:

- они несут hidden context debt;
- могут вернуть старый drift;
- смешивают Need, Concept, Architecture, MVP, Execution;
- могут снова превратить Project в продолжение старого длинного чата.

## Как использовать старые чаты

Старые чаты использовать только как:

`Reference Archive`  
(справочный архив)

а не как:

`Source-of-truth`  
(источник истины)

## Когда можно возвращаться к старым чатам

Только если:

1. в новом Project возник конкретный пробел;
2. пробел связан с конкретной фазой;
3. есть целевой Recovery Prompt;
4. результат будет извлечён как артефакт;
5. старый чат не будет импортирован целиком.

## Формат работы со старым чатом

Правильно:

```text
Новый Project выявил gap
↓
формируем точный Recovery Prompt
↓
прогоняем в старом чате
↓
получаем артефакт
↓
проверяем
↓
импортируем или паркуем
```

Неправильно:

```text
загрузить старый чат
↓
продолжить оттуда
```

## На старте нового Project

Не нужны:

- исходный старый чат;
- все промежуточные диалоги;
- длинные рассуждения;
- raw transcripts.

Нужны:

- Starter Pack v0.2;
- Knowledge Recovery Pack v0.1;
- Wave 3 Closure Pack v0.1;
- Project Initialization File.
