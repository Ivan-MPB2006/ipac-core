# 4. Что изменилось для IPaC

До этого:

```text
тред был огромным потоком;
часть смысла терялась;
контекст было трудно вернуть в работу;
summary не заменяло первичку.
```

Теперь:

```text
тред становится первичным месторождением;
Obsidian/PDF/MD становятся контейнерами нулевого слоя;
Git даёт версионирование;
diff даёт фактографию дня;
выкопировка даёт повторный вход контекста;
Daily Trace фиксирует смысл;
реестры дают дальнейшее использование.
```

# 5. Поисковое проектирование в действии

Этот комплект создан в режиме:

```text
Exploratory Design by Crawling
(поисковое проектирование через личное проползание)
```

Формула:

```text
Не проектируем сверху по красивой схеме.

Сначала проползаем на пузе,
задеваем все коряги,
чувствуем путь шкурой,
фиксируем трение,
а потом превращаем найденный путь
в правило, техпроцесс, регламент и схему.
```

Это важный метод IPaC:

```text
личное прохождение → фактография → осмысление → правило → Git → повторяемая технология.
```
