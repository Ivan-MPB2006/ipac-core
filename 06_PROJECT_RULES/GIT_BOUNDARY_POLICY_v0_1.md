# Git Boundary Policy v0.1

## What should be versioned

- canon markdown
- registers
- decision records
- templates
- maps

## What should not be versioned without review

- private raw chats
- temporary exports
- accidental screenshots
- secrets / tokens
- unreviewed dumps

## Acceptable unit of change

- one reviewed artifact change
- one decision record
- one register update
- one controlled canon rebuild

## Forbidden

- silent canon overwrite
- `latest` file replacing versioned canon
- execution branch before explicit mode change
