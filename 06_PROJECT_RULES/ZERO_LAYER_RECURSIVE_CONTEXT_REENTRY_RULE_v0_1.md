# 0. НКУ / Новый качественный уровень

```text
НКУ-2026-06-20-001

Переход от “чатимся в никуда”
к многопрогонной системе сохранения, добычи,
повторного входа и усиления контекста.
```

Суть НКУ:

```text
Контекст не исчезает после разговора.

Он отчуждается в Obsidian / PDF / MD,
сохраняется как нулевой слой,
выборочно добывается как тематический пласт,
возвращается в канал обратной связи
и перерабатывается на новом уровне развития.
```

Это означает, что тред перестаёт быть одноразовой коммуникацией.  
Он становится сырьевым пластом для последующих проходов.

# 1. Главная формула

```text
Raw Thread
(сырой тред)

→ Source Container
(контейнер первоисточника)

→ Selective Extraction
(выборочная выкопировка)

→ Feedback Channel
(канал обратной связи)

→ Higher-Level Reprocessing
(переработка на новом уровне)

→ Artifact / Rule / Register
(артефакт / правило / реестр)

→ Git Commit
(Git-проводка)

→ New Save Area Checkpoint
(новая контрольная точка Области Сохранения)
```

Коротко:

```text
Нулевой слой — не склад прошлого.

Нулевой слой — сырьевой пласт,
из которого контекст возвращается в работу
и усиливает следующий проход.
```

---

# 2. Почему это критично

До появления этого контура чат имел риск превращаться в пыль:

```text
поговорили;
получили смысл;
контекст ушёл;
тред разросся;
модель забыла;
следующий проход начинается почти заново.
```

После появления контура:

```text
поговорили;
сохранили;
отчуждили;
нашли нужный пласт;
вернули на вход;
переработали на новом уровне;
зафиксировали в артефакт;
провели через Git.
```

Это превращает IPaC в многопрогонную систему постоянного улучшения.

# 3. Primary-source principle / принцип первички

Обобщённая память недостаточна.

Работать нужно не только по пересказу, summary (саммари) или “вроде бы мы это обсуждали”, а по первичным документам:

```text
выкопировка из треда;
PDF / MD экспорт;
Obsidian source container;
thread export;
Daily Register;
Daily Trace;
commit-check report;
Live State Snapshot.
```

Принцип:

```text
Primary source first.
(сначала первоисточник)

Summary second.
(обобщение — вторично)

Interpretation third.
(интерпретация — после первички)
```

Обобщение полезно, но оно не заменяет первичный слой.

---

# 4. Ассоциативный доступ к первичке

Первичка должна быть не просто “где-то сохранена”.

Она должна быть доступна на ассоциативном уровне через Obsidian:

```text
ссылки;
backlinks (обратные ссылки);
теги;
MOC / Map of Content (карта содержания);
Daily Trace;
регистры;
source-container notes;
routing maps;
decision records;
thread export manifests.
```

Ключевая формула:

```text
Наличие первички без ассоциативного доступа
= архив.

Первичка + ассоциативный доступ
= рабочая память смысловой ОС.
```

# 5. Правило повторного входа контекста

Если новый этап работы опирается на старый большой тред, запрещено работать только “по памяти”.

Нужно:

```text
1. Найти первичный источник.
2. Сделать тематическую выкопировку.
3. Указать дату / тред / источник.
4. Вернуть фрагмент в рабочий канал.
5. Обновить инструкцию / процесс с учётом текущего состояния.
6. Зафиксировать результат в Daily Register / Daily Trace.
7. Провести значимые артефакты через Git.
```

# 9. Критерии качества выкопировки

Хорошая выкопировка должна содержать:

```text
1. дату исходного фрагмента;
2. указание источника;
3. сохранение исходного хода мысли;
4. минимальное вмешательство в первичный текст;
5. явное отделение первички от интерпретации;
6. цель повторного входа;
7. какие решения / артефакты она должна обновить;
8. ссылку на Source Container;
9. статус актуальности:
   актуально / устарело / требует обновления / требует проверки.
```

---

# 10. Правило актуализации

Выкопировка из старого треда не считается автоматически актуальной.

Её нужно пропустить через актуализацию:

```text
1. Что ещё актуально?
2. Что устарело?
3. Что заменено новыми правилами?
4. Что усилено новыми артефактами?
5. Что требует новой проверки?
6. Что стало частью канона / правила / Daily Trace?
```

Пример:

```text
Выкопировка 18.06 по Project IPaC NIR сохранила ядро:
Resource Pack + Live State Snapshot + Bootloader Prompt.

Но состояние проверки устарело,
потому что 19.06 появились:
- Operational Frame v0.2;
- File-Based Operational Factography Rule;
- Save Area Checkpoint insight;
- свежий Live State Snapshot.
```
