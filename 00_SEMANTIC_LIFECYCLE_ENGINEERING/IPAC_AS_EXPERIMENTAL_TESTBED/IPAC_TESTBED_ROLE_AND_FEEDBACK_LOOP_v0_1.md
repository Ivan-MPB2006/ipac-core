```yaml

artifact_type: testbed_positioning

```

## 1. Позиция IPaC

IPaC не является всем подходом Semantic Lifecycle Engineering.

IPaC — это полигон, на котором:

```text
проверяются концептуально-архитектурные решения;
строятся операционные правила;
отрабатываются Git / Obsidian / Project-процессы;
проверяется слой 0;
проверяется Daily Register / Ship Log;
проверяется Daily Trace;
формируется обратная связь наверх.
```

## 2. Что IPaC уже апробирует

```text
Layer 0 / Factography
(слой 0 / фактография)

Dual Memory Architecture
(двойная архитектура памяти): Obsidian + Git

Ship Log
(Бортовой журнал)

Daily Trace
(Дневная трасса)

Project Supervisor
(Project-супервизор)

Live State Snapshot
(снимок живого состояния)

Save–Transfer–Restore Area
(Область Сохранения — Передачи — Восстановления)

External Resource Management
(управление внешними ресурсами) через будущий Codex/MCP
```

## 3. Обратная связь от IPaC наверх

Каждое сильное событие IPaC может быть маршрутизировано как:

```text
concept_validation_feedback
(обратная связь о проверке концепта)

architecture_feedback
(архитектурная обратная связь)

process_feedback
(процессная обратная связь)

tooling_feedback
(обратная связь по инструментам)

risk_feedback
(обратная связь по рискам)

methodological_feedback
(методологическая обратная связь)
```

## 4. Правило

```text
IPaC не поглощает верхнеуровневый подход.
IPaC проверяет его элементы и возвращает результаты апробации.
```
