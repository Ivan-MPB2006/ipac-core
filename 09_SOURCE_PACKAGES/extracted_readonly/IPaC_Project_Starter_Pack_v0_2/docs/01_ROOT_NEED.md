# 01 — Root Need

## Фундаментальная потребность

```text
Создать инженерно управляемый способ производства, оформления, проверки, развития и оборота живого знания,
который позволяет сложному человеческому замыслу проходить путь
от интуитивного смыслового импульса
до отчуждаемых артефактов, исполнения, feedback, review, canon update и trace,
не распадаясь, не превращаясь в хаотичный документооборот
и не теряя живую смысловую целостность.
```

## Короткие формулы

```text
Нужна не система документов.
Нужна система оборота живого знания.
```

```text
Сложный смысл должен становиться исполнимым знанием.
```

```text
Output стал дешёвым, смысл остался дорогим.
```

English:

```text
AI made output cheap. Now meaning is the bottleneck.
```

## Для кого существует проблема

1. Для автора / носителя сложного замысла.
2. Для проектной системы, которая быстро производит артефакты.
3. Для организаций, где AI-output растёт быстрее meaning production.
4. Для будущих Project, ИИ, Codex, агентов и участников, которым нужен handoff не только материалов, но и способа работы.

## Главные боли

- сложный смысл распадается при переносе в документы;
- контекст теряется между чатами, тарифами, Project, агентами и инструментами;
- архитектура может стать способом отложить действие;
- feedback легко превращается в ложную валидацию;
- каналы публикации могут начать управлять мышлением;
- документооборот не равен обороту живого знания.

## Что НЕ является решением само по себе

- обычный документооборот;
- knowledge management без живого контура feedback/review/trace;
- контент-план;
- набор публикационных каналов;
- длинный чат;
- отдельная модель ИИ;
- красивый summary.

## Статус

Root Need принят как `accepted-candidate` и является главным потребностным инвариантом Starter Pack v0.1.
