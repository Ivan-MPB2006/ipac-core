# 5. Boundary Interpretation

## 5.1 What IPaC inherits

IPaC явно наследует и пересекается с:

- Knowledge Engineering;
- Knowledge Acquisition;
- Knowledge Representation;
- Ontology Engineering;
- Knowledge Graph Engineering;
- Knowledge Management;
- Provenance Engineering;
- Docs-as-Code / DocOps;
- ADR / Configuration Management;
- Human-in-the-loop AI;
- Cognitive Systems Engineering;
- Context Engineering;
- Decision Support Systems.

## 5.2 What differentiates IPaC

Наиболее сильная зона IPaC — не отдельные термины, а сборка операционного контура:

```text
pre-formal sense capture
→ knowledge clump
→ knowledge artifact
→ canon
→ trace
→ feedback / evidence
→ review
→ decision
→ controlled rebuild
→ updated canon
```

Отличие IPaC следует искать не в том, что он изобрёл каждый отдельный элемент, а в том, что он связывает их в явный human-AI operating contour.

# 6. Claims to Weaken

Следующие формулировки должны быть ослаблены:

| Claim | Why Weaken | Safer Formulation |
|---|---|---|
| IPaC is a new discipline separate from KE | Высокое наследование от KE / KM / ontology / provenance. | IPaC is an integrative operating framework with strong KE overlap. |
| IPaC invented multi-layer knowledge representation | Слоевые модели есть в CommonKADS / ontology engineering. | IPaC adapts layered representation to local-first human-AI workflow. |
| Canon / trace / controlled rebuild are new concepts | Есть аналоги: source-of-truth, provenance, maintenance, configuration management. | IPaC делает эти практики явными и связанными в одном контуре. |
| Living Meaning is already validated | Нет field validation. | Living Meaning is a candidate pre-formal layer requiring validation. |
| Research support validates IPaC | Research support ≠ field validation. | Research clarifies positioning; validation requires field tests. |

---

# 7. Claims to Strengthen

Следующие формулировки можно усиливать осторожно:

| Claim | Reason | Suggested Formulation |
|---|---|---|
| Integrative operating discipline | Оба DR-отчёта сходятся в этом направлении. | IPaC is primarily an integrative operating discipline, not a new representation formalism. |
| Pre-formal sense capture | Living Meaning / Knowledge Clump вынесены как явный слой. | IPaC foregrounds the transition from pre-formal meaning to managed artifacts. |
| Governance / trace / review | Контур review → decision → trace → rebuild явно операционализирован. | IPaC makes provenance-like governance a first-class element of knowledge work. |
| Human-AI workflow | Современная KE смещается к LLM / KG / neuro-symbolic workflows. | IPaC is an AI-native workflow hypothesis built on KE/KM/provenance inheritance. |
| Generalized information blocks | IPaC потенциально шире DocOps. | IPaC may apply beyond technical documentation to research, communication and decision-support artifacts. |
