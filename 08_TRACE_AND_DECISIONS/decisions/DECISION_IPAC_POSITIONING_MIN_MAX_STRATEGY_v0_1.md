# 2. Minimum Defensible Position

Минимально защищаемая позиция:

```text
IPaC — это human-AI framework
для дисциплинированного производства,
сопровождения и обновления knowledge artifacts
через Obsidian, Git, review, decision, trace и controlled rebuild.
```

Эта позиция остаётся устойчивой даже если значительная часть элементов IPaC окажется уже известной в Knowledge Engineering, Knowledge Management, Docs-as-Code, Provenance Engineering, ADR / Configuration Management или соседних дисциплинах.

---

# 3. Maximum Research Hypothesis

Максимальная исследовательская гипотеза:

```text
IPaC — это вход в Инженерию информационного производства:
более широкую область, которая проектирует производство,
управление, валидацию, версионирование и переиспользование
информационных блоков для человеческого понимания,
AI-обработки, коммуникации, исследования и принятия решений.
```

Эта гипотеза НЕ считается доказанной.

Она должна проверяться через:

- DR-001 OpenAI Deep Research report;
- DR-001 Gemini report;
- DR-001P Perplexity source triangulation;
- comparative / triangulated research review;
- decision record;
- field validation later.

---

# 4. Strategic Rationale

IPaC не должен защищаться через искусственное утверждение радикальной новизны.

Сильная позиция IPaC может состоять не в изобретении каждого отдельного элемента, а в сборке операционного контура:

```text
pre-formal sense capture
→ knowledge artifact
→ canon
→ trace
→ feedback / evidence
→ review
→ decision
→ controlled rebuild
→ updated canon
```

При этом IPaC должен честно признавать наследование и пересечения с:

- Knowledge Engineering;
- Knowledge Management;
- Ontology Engineering;
- Knowledge Graph Engineering;
- Provenance Engineering / PROV-O;
- Docs-as-Code;
- ADR / Configuration Management;
- Cognitive Systems Engineering;
- Context Engineering;
- Human-in-the-loop AI.
